CASR is a toolkit for security engineers and fuzzing practitioners who need to turn a flood of raw crashes into actionable findings. It generates structured `.casrep` reports (JSON) containing severity ratings, stack traces, register values, disassembly, and source context — then deduplicates and clusters those reports so you focus on unique bugs, not duplicates.

## How CASR Works

<Steps>
  <Step title="Collect crash reports">
    Run one of the `casr-*` tools against your crashing binary, sanitizer output, or coredump. CASR captures the full execution context and writes a `.casrep` file.
  </Step>
  <Step title="Deduplicate">
    Use `casr-cluster -d` to remove duplicate reports based on filtered stack trace hashing, leaving only unique crashes.
  </Step>
  <Step title="Cluster">
    Run `casr-cluster -c` to group related crashes together. Each cluster represents a distinct root cause.
  </Step>
  <Step title="Triage and act">
    View reports interactively with `casr-cli`, export them as SARIF for IDE integration, or upload to DefectDojo with `casr-dojo`.
  </Step>
</Steps>

## What CASR Covers

<CardGroup cols={3}>
  <Card title="Sanitizers" icon="shield">
    AddressSanitizer, MemorySanitizer, and UndefinedBehaviorSanitizer via `casr-san` and `casr-ubsan`.
  </Card>
  <Card title="Coredumps" icon="circle-xmark">
    Offline and online (kernel core_pattern) coredump analysis via `casr-core`.
  </Card>
  <Card title="GDB" icon="magnifying-glass">
    Raw GDB-based crash capture for uninstrumented binaries via `casr-gdb`.
  </Card>
  <Card title="8 Languages" icon="globe">
    C/C++, Rust, Python, Java, JavaScript, C#, Go, and Lua crash parsing.
  </Card>
  <Card title="Fuzzer Pipelines" icon="arrows-rotate">
    End-to-end triage automation for AFL++, libFuzzer, LibAFL, Jazzer, Atheris, Jazzer.js, jsfuzz, and luzer.
  </Card>
  <Card title="SARIF & DefectDojo" icon="chart-bar">
    Export to SARIF for IDE integration or upload to DefectDojo for vulnerability management.
  </Card>
</CardGroup>

<Note>
  CASR requires `--cap-add=SYS_PTRACE --security-opt seccomp=unconfined` when running inside Docker, because it disables address randomization and uses ptrace to run GDB.
</Note>
